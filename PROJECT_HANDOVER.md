# MedGraphics Engine - Project Handover

Welcome to the MedGraphics Engine! This document serves as a comprehensive handover guide for new engineers joining the project.

## 🎯 Project Overview
MedGraphics Engine is an automated AI-driven pipeline that generates visually stunning, pixel-perfect medical infographics. It parses user requests, queries LLMs for medically accurate topic curation, structures the data into JSON, injects it into Jinja2 HTML/CSS templates, and renders the final result into high-resolution PNGs using a headless Chromium browser.

All of this is wrapped in an interactive, 4-page Streamlit web GUI.

## 🛠️ Tech Stack & Architecture
- **Language**: Python 3.12+
- **LLM Orchestration**: `LiteLLM` (Handles fallback routing between Gemini, OpenAI, and Anthropic seamlessly).
- **Frontend / UI**: `Streamlit` (Provides a lightweight, reactive state-driven web dashboard).
- **Templating**: `Jinja2` (HTML/CSS string interpolation).
- **Rendering Engine**: `Playwright` (Headless browser rendering raw HTML strings into PNG buffers).
- **Testing**: `PyTest` / `unittest` (End-to-end and component tests executed natively).

## 🚀 Getting Started for Engineers
1. Clone the repository and `git checkout dev`.
2. Install dependencies: `pip install -r requirements.txt`.
3. Install Playwright binaries: `playwright install chromium --with-deps`.
4. Run locally: `streamlit run app.py`.
5. Enter your API credentials in the "Model Settings" tab directly.
6. Verify development sanity by executing: `python -m pytest tests/`.

**Note:** If API keys are missing on local execution, the `test_pipeline.py` and `test_llm.py` tests will safely execute a `pytest.skip()` rather than bombing your terminal red.
